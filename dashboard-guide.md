# Dashboard UX Design Guide

BubbleShare 대시보드 설계 시 지켜야 할 원칙과 방법론.

---

## 1. 정보 계층 (Information Hierarchy)

### 시선 흐름을 따라라
- 사람은 F/Z 패턴으로 화면을 읽는다. **가장 중요한 숫자는 왼쪽 위**에.
- 위에서 아래로: 전체 요약 → 개요 → 상세 분석 순서.
- 오른쪽으로 갈수록, 아래로 갈수록 주의력이 떨어진다.

### 3초 룰
- 대시보드를 열었을 때 **3초 안에 전체 상황을 파악**할 수 있어야 한다.
- 핵심 숫자는 크고 굵게. 보조 정보는 작고 연하게.
- "모든 걸 보여주는 것"이 아니라 "판단에 필요한 것만 보여주는 것"이 목표.

### 타이포그래피로 계층을 만들어라
- 핵심 지표: 큰 글씨 + 볼드
- 보조 지표: 중간 크기
- 라벨/설명: 작은 크기 + 연한 색
- 색상이 아니라 **크기와 굵기**로 중요도를 전달.

---

## 2. 메트릭 구성 (Metric Organization)

### 1차값과 2차값을 구분하라
- **1차값 (Input)**: 직접 발생한 원본 데이터 (매출, 비용, 목표, 투입시간)
- **2차값 (Derived)**: 1차값을 계산해서 나온 값 (마진, 달성률, CM/Hour)
- 2차값을 보여줄 때는 **어떤 1차값에서 나왔는지** 공식을 함께 표시.

### 관련 지표는 묶어라
- 독립적인 숫자 나열 ✕
- 인과관계가 있는 지표를 가까이 배치 ○
- 예: 매출 옆에 비용, 그 결과로 마진 — 셋이 함께 있어야 관계가 보인다.

### 모든 지표에 맥락을 붙여라
- 숫자만 있으면 의미 없다. 반드시 비교 기준을 함께.
- 비교 대상: 목표, 전기 대비, 전월 대비, 평균
- "12.7억"보다 "12.7억 (목표 대비 +54%)"가 판단 가능.

---

## 3. Progressive Disclosure (단계적 공개)

### 개요 먼저, 상세는 요청 시
- 첫 화면: 핵심 숫자 2-3개 + 차트
- 탭/필터 전환: 다른 관점으로 같은 데이터를 봄
- 클릭/드릴다운: 특정 항목의 상세로 이동

### 같은 공간에서 전환하라
- 페이지를 늘리지 말고, **탭/필터/토글**로 같은 영역에서 다른 정보를 보여줘라.
- 사용자가 맥락을 잃지 않으면서 탐색할 수 있다.
- 카드 5개보다 탭 3개 + 카드 2개가 낫다.

### 호버/툴팁으로 보조 정보를 제공하라
- 기본 화면을 깔끔하게 유지하면서 추가 설명은 호버 시 표시.
- 전문 용어, 약어, 공식은 툴팁으로 설명.
- "CM/Hour"에 호버하면 "공헌이익 / 총 투입시간 = 시간당 벌어들이는 이익" 표시.

---

## 4. 차트 원칙 (Chart Rules)

### 차트 타입 선택
- **시간 추이**: 라인 차트 (트렌드를 보여줄 때)
- **항목 비교**: 바 차트 (크기를 비교할 때)
- **구성 비율**: 원형/도넛 (전체 대비 비중)
- **관계**: 스캐터 플롯 (두 변수의 상관관계)
- 목적에 맞지 않는 차트 타입은 데이터를 왜곡한다.

### 차트를 깨끗하게 유지하라
- 데이터 시리즈는 최대 4-5개. 그 이상이면 필터로 분리.
- Y축은 반드시 0에서 시작 (비율 왜곡 방지).
- 범례는 일관된 위치에 (보통 하단 중앙).
- 그리드라인은 연하게 (데이터보다 눈에 띄면 안 됨).

### 실적과 전망을 시각적으로 구분하라
- 실적(확정 데이터): 실선, 진한 색
- 전망(예측 데이터): 점선, 연한 색
- 이 구분은 색상+패턴 둘 다 사용 (색맹 접근성).

---

## 5. 색상 전략 (Color Strategy)

### 그레이스케일을 기본으로
- 명도(밝기)로 정보 계층을 만든다.
- 진한 색 = 중요/실적, 연한 색 = 보조/전망/기준선
- 색상은 **예외 상황에만** 사용 (목표 미달, 경고, 달성).

### 빨강-초록 신호등을 피하라
- 색맹 사용자가 구분 못 함.
- 색상 + 아이콘/패턴/텍스트를 함께 사용.
- 예: 빨간색 + ↓ 아이콘 + "미달" 텍스트

### 색상 일관성
- 같은 개념에는 같은 색. 대시보드 전체에서 통일.
- 매출=A색, 비용=B색이면 모든 페이지에서 동일하게.

---

## 6. 인터랙션 패턴 (Interaction Patterns)

### 일관성
- 같은 인터랙션 패턴을 모든 섹션에서 사용.
- 한 곳에서 탭으로 전환하면, 다른 곳도 탭으로 전환.
- 사용자가 한번 배우면 어디서든 적용 가능해야 함.

### 필터는 목적에 맞게
- 페이지 전체에 영향: 상단 글로벌 필터 (기간, 고객 등)
- 특정 차트에만 영향: 차트 내 로컬 필터
- 필터 적용 중 로딩 피드백 필수.

### 드릴다운은 보상이 있어야 한다
- 클릭했을 때 "더 깊은 인사이트"가 나와야 함.
- 같은 정보를 다른 형태로 보여주는 건 드릴다운이 아님.
- 개요 → 상세로 이동할 때 맥락을 유지 (뒤로가기 링크 등).

---

## 7. 재무 대시보드 특화 규칙

### 관계 지표는 나란히 배치
- 고객 획득 비용(CAC) ↔ 고객 생애 가치(LTV)
- 매출 ↔ 비용 ↔ 마진
- 목표 ↔ 실적 ↔ 차이

### 파생 지표에 공식을 붙여라
- 공헌이익: "매출 12.7억 - 비용 1.9억"이 함께 보여야 함.
- 복잡한 계산에는 주석(annotation) 추가.
- 사용자가 "이 숫자가 어디서 나왔지?"를 스스로 검증할 수 있어야 함.

### 표(Table)는 정밀 데이터에 사용
- P&L 상세 데이터처럼 정확한 수치가 필요할 때.
- 차트는 트렌드와 비교에, 표는 정확한 값 확인에.

## 8. 피해야 할 실수 (Anti-Patterns)

| 실수 | 왜 문제인가 | 대안 |
|------|-----------|------|
| 있으니까 다 보여주기 | 인지 과부하, 판단 불가 | 핵심만 기본 표시, 나머지는 인터랙션으로 |
| 맥락 없는 숫자 | "12.7억"만으로는 좋은지 나쁜지 모름 | 목표/전기 대비 비교 함께 표시 |
| 카드 수 늘리기로 해결 | 공간만 차지, 관계 안 보임 | 탭/필터로 같은 공간에서 전환 |
| 전문 용어 설명 없음 | 비전문가 접근 불가 | 툴팁/주석으로 설명 |
| 차트에만 의존 | 정확한 값 확인 어려움 | 차트 + 핵심 숫자 병행 |
| 색상으로만 구분 | 색맹 접근성, 인쇄 시 구분 불가 | 색상 + 패턴 + 텍스트 조합 |
| 모바일 무시 | 이동 중 확인 불가 | 모바일은 최우선 지표만 표시 |

## 9. 카드 설계 원칙 (PatternFly 기반)

### 하나의 카드 = 하나의 메트릭 (또는 밀접한 관련 메트릭 그룹)
- 카드에 무관한 정보를 섞지 않는다.
- 카드 크기: S(단일 숫자), M(차트 포함), L(테이블 포함)

### 카드 우선순위로 배치
1. 가장 중요한 카드를 상단에
2. 중요도가 같으면 왼쪽에서 오른쪽으로
3. 동적 컨텐츠 카드는 확장 공간을 고려

### 카드 유형
- **집계 카드**: 전체 수량 + 상태별 분류 (예: 고객 54개, 활성 42개)
- **트렌드 카드**: 현재 값 + 스파크라인 (예: 매출 12.7억 + 월별 미니 차트)
- **상세 카드**: 속성-값 목록 (예: 항목별 비용 리스트)

## 10. 이 프로젝트에 적용

### 홈 페이지
- 핵심 2차값을 탭으로 전환 (수익성/목표/효율성)
- 선택된 2차값 크게 표시 + 아래에 1차값 공식 분해
- 차트는 선택된 맥락에 맞는 것만 표시

### 매출 페이지
- 필터 기반 단일 차트 (고객/패키지 × 매출/비용/마진 × 비교/추이) ✅ 완료

### 비용 페이지
- 금액 순 랭킹 리스트 + 시간 투입 ✅ 완료

### 목표 페이지
- CM/Hour 단일 지표 집중 + 효율화 아이템 ✅ 완료
